Welcome to IoTFrog project

The objecive of this project is study of develop a neuronal IoT network, based on ESP-8266, the main requisites are:

  • Easy for end users
  • Only need config same domain and password
  • Remote managment using web statdard browsed (HTML5, CSS3, javascrit)
  • Nothing to intall on remote manager device (Smatphone, PC,...)
  • Reply of devices as Html and json

You can see our web site on: using www.IoTfrog.com

Template of Modules requisites.

Standar petition: All modules has a web server and they reply to this url

  • / -> Reply html form with menu and short descrition
  • who -> (Who) reply de the name of device by default ESP-(ID)
  • whe -> (Where) Reply where is it
  • wha -> (What) Reply a list of orders and descrition of his reply
  • sts -> (Status) Reply a list json of the status of sensor and actuattors.
  • aba -> (Abaut) Short descrition and links for help and support
  • hlp -> (help) Reply a short guide
  • log -> (Loggin) Reply login form (Need for to do changes)
  • tre -> (Tree) Reply description id for build a tree display (treeId, treeParentId, treeclass, and who)
  • cfgw -> (Config wifi) send html form for config SIID, Password, IP,Mask, ...
  • cfgo -> (Config other) send html form for config especific parameter of module, like email for send alarmas, time values, start stop schendule... Specifig module petition
  • mod_xxx ->(Module specific) this is the sintax prefig for petition for set specific module values.

Examples: mod_led?id=1&val=1 -> call handleLed method for set led 1 on, the reply say {led:1,val=1} mod_led?id=1&val=0 -> call handleLed method for set led 1 off mod_led?id=1 -> call handleLed method for get status of led 1, the reply can be {led:1,val=1} mod_temp?id=1 -> call handleTemp method for get temperature of sensor 1, the reply can be {temp:1,val:27.05,unit:C}

set_relay?id=1&val=1 -> call handleRelay method for set relay 1 on

Note: All pettition can be asqued as html or json.
